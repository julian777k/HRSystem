## 1. 관리자 권한 구조

MSA HR 시스템은 4단계의 권한 체계를 갖추고 있습니다.

| 권한 | 코드 | 설명 |
|------|------|------|
| 시스템 관리자 | SYSTEM_ADMIN | 모든 기능 접근 가능. Setup Wizard에서 생성한 최초 관리자 |
| 회사 관리자 | COMPANY_ADMIN | 전사 인사 관리 (직원 등록, 연차 부여, 규정 설정 등) |
| 부서 관리자 | DEPT_ADMIN | 소속 부서의 직원 관리 및 결재 처리 |
| 일반 사용자 | BASIC | 본인의 휴가 신청, 현황 조회 |

### 권한별 접근 가능 메뉴

| 복지신청 | O | O | O | O |
| 복지관리 | O | O | - | - |
| 직원관리 | O | O | - | - |
| 권한/결재선 설정 | O | O | - | - |

## 2. 직원 관리

좌측 메뉴에서 **기본설정 > 직원관리** 를 클릭합니다.

### 2-1. 직원 등록 (신규 입사)

1. **"직원 등록"** 버튼을 클릭합니다.

2. 아래 항목을 입력합니다.

| 항목 | 필수 | 설명 | 예시 |
|------|:---:|------|------|
| 사번 | O | 고유 사원번호 | 2026001 |
| 이름 | O | 직원 이름 | 홍길동 |
| 이메일 | O | 로그인용 이메일 (중복 불가) | hong@company.com |
| 초기 비밀번호 | O | 8자 이상, 첫 로그인 시 변경 유도 | temp1234! |
| 전화번호 | - | 연락처 | 010-1234-5678 |
| 부서 | O | 드롭다운에서 선택 | 개발팀 |
| 직급 | O | 드롭다운에서 선택 | 대리 |
| 입사일 | O | 날짜 선택 | 2026-03-01 |
| 시스템 권한 | O | 권한 수준 선택 | 일반 사용자 |

3. **"저장"** 버튼을 클릭합니다.

4. 직원이 등록되면 자동으로 입사일 기준 연차가 계산 및 부여됩니다.

> **중요**: 직원에게 초기 비밀번호와 시스템 접속 주소를 안내해 주세요. 첫 로그인 후 비밀번호를 변경하도록 안내하세요.

### 2-2. 직원 정보 수정

1. 직원 목록에서 수정할 직원의 행을 클릭하거나 **"수정"** 버튼을 클릭합니다.

2. 수정이 필요한 항목을 변경합니다.
   - 부서 이동, 직급 변경, 연락처 변경 등

3. **"저장"** 버튼을 클릭합니다.

> **참고**: 사번과 이메일은 변경 시 중복 여부가 자동으로 검사됩니다.

### 2-3. 직원 퇴사 처리

1. 직원 목록에서 퇴사할 직원을 선택합니다.

2. **"퇴사 처리"** 버튼을 클릭합니다.

3. 퇴사일을 입력합니다.

4. **"확인"** 을 클릭합니다.

퇴사 처리 시 자동으로 수행되는 작업:
- 직원 상태가 **RESIGNED(퇴사)** 으로 변경됩니다.
- 해당 직원의 로그인이 비활성화됩니다.
- 미처리된 휴가 신청이 자동 취소됩니다.
- 잔여 연차 정보가 보존됩니다 (조회 가능).

> **참고**: 퇴사 처리된 직원의 데이터는 삭제되지 않으며, 이력 조회가 가능합니다.

### 2-4. 직원 검색 및 필터

직원 목록 상단의 검색/필터 기능을 활용할 수 있습니다.

| 필터 | 설명 |
|------|------|
| 이름/사번 검색 | 이름 또는 사번으로 검색 |
| 부서 필터 | 특정 부서 직원만 표시 |
| 직급 필터 | 특정 직급 직원만 표시 |
| 상태 필터 | 재직/휴직/퇴사 직원 필터링 |

## 3. 직원 엑셀 일괄 등록

다수의 직원을 한 번에 등록할 때 엑셀 파일을 사용합니다.

### 3-1. 엑셀 템플릿 다운로드

1. **직원관리** 페이지에서 **"엑셀 양식 다운로드"** 버튼을 클릭합니다.

2. 아래와 같은 컬럼이 포함된 엑셀 파일이 다운로드됩니다.

| 열 | 필수 | 설명 | 예시 |
|----|:---:|------|------|
| A: 사번 | O | 고유 사원번호 | 2026001 |
| B: 이름 | O | 직원 이름 | 홍길동 |
| C: 이메일 | O | 로그인용 이메일 | hong@company.com |
| D: 초기비밀번호 | O | 8자 이상 | temp1234! |
| E: 전화번호 | - | 연락처 | 010-1234-5678 |
| F: 부서 | O | 부서명 (정확히 입력) | 개발팀 |
| G: 직급 | O | 직급명 (정확히 입력) | 대리 |
| H: 입사일 | O | YYYY-MM-DD 형식 | 2026-03-01 |

### 3-2. 엑셀 파일 작성 주의사항

- **부서명과 직급명은 시스템에 등록된 이름과 정확히 일치**해야 합니다.
  - 예: "개발팀"으로 등록되어 있으면 "개발 팀"이나 "개발부"로 입력하면 오류 발생
- **이메일은 중복 불가**입니다. 이미 등록된 이메일은 오류 처리됩니다.
- **사번도 중복 불가**입니다.
- 입사일은 반드시 **YYYY-MM-DD** 형식으로 입력합니다.
- 비밀번호는 반드시 8자 이상이어야 합니다.
- 첫 번째 행은 헤더이므로 데이터는 2행부터 입력합니다.

### 3-3. 엑셀 파일 업로드

1. **"엑셀 업로드"** 버튼을 클릭합니다.

2. 작성한 엑셀 파일을 선택합니다.

3. 업로드 전 데이터 미리보기가 표시됩니다.
   - 오류가 있는 행은 빨간색으로 표시됩니다.
   - 오류 내용(예: "부서명이 존재하지 않습니다")이 표시됩니다.

4. 오류가 없으면 **"등록"** 버튼을 클릭합니다.

5. 등록 결과가 표시됩니다 (성공 건수, 실패 건수).

### 3-4. 직원 목록 엑셀 내보내기 (Export)

1. **"엑셀 다운로드"** 버튼을 클릭합니다.

2. 현재 필터 조건에 맞는 직원 목록이 엑셀 파일로 다운로드됩니다.

## 4. 부서 관리

### 4-1. 부서 등록

1. **직원관리** 페이지 내 부서 관리 영역에서 **"부서 추가"** 버튼을 클릭합니다.

2. 아래 항목을 입력합니다.

| 항목 | 필수 | 설명 | 예시 |
|------|:---:|------|------|
| 부서명 | O | 부서 이름 (중복 불가) | 개발팀 |
| 부서 코드 | O | 영문 코드 (중복 불가) | DEV |
| 상위 부서 | - | 트리 구조에서 상위 부서 | 기술본부 |
| 정렬 순서 | - | 목록 표시 순서 | 1 |

3. **"저장"** 버튼을 클릭합니다.

### 4-2. 부서 트리 구조

MSA HR은 부서의 트리(계층) 구조를 지원합니다.

예시:
```
회사 전체
├── 경영지원본부
│   ├── 경영지원팀
│   ├── 인사팀
│   └── 재무팀
├── 기술본부
│   ├── 개발팀
│   ├── 인프라팀
│   └── QA팀
└── 영업본부
    ├── 국내영업팀
    └── 해외영업팀
```

### 4-3. 부서 수정 및 삭제

- **수정**: 부서명, 코드, 상위 부서, 정렬 순서를 변경할 수 있습니다.
- **비활성화**: 더 이상 사용하지 않는 부서를 비활성화합니다. 해당 부서에 소속된 직원이 있으면 먼저 다른 부서로 이동해야 합니다.

> **주의**: 부서에 소속된 직원이 있는 경우 부서를 삭제할 수 없습니다. 먼저 직원을 다른 부서로 이동하세요.

## 5. 직급 관리

시스템에는 기본 직급이 사전 등록되어 있습니다.

### 기본 직급 체계

| 직급 | 레벨 | 설명 |
|------|------|------|
| 이사 | 1 | 최상위 직급 |
| 상무 | 2 | |
| 부장 | 3 | |
| 차장 | 4 | |
| 과장 | 5 | |
| 대리 | 6 | |
| 사원 | 7 | 최하위 직급 |

레벨 값이 낮을수록 높은 직급입니다. 결재선에서 상위 직급자를 자동 지정할 때 이 레벨 값이 사용됩니다.

### 직급 추가/수정

1. 직급 관리 영역에서 **"직급 추가"** 또는 기존 직급의 **"수정"** 버튼을 클릭합니다.

2. 직급명과 레벨을 입력합니다.
   - 레벨은 중복될 수 없습니다.
   - 새로운 직급을 추가할 때 기존 레벨 사이에 삽입할 수 있습니다.

3. **"저장"** 버튼을 클릭합니다.

## 6. 휴가 부여 및 차감

좌측 메뉴에서 **휴가 > 휴가부여** 를 클릭합니다.

### 6-1. 연차 수동 부여

근속연수에 따른 자동 부여 외에, 특별 연차를 수동으로 부여할 수 있습니다.

1. **"연차 부여"** 버튼을 클릭합니다.

2. 아래 항목을 입력합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| 대상 직원 | 검색하여 선택 | 홍길동 (2026001) |
| 휴가 종류 | 연차, 특별휴가 등 | 연차휴가 |
| 부여 일수 | 부여할 일수 | 3 |
| 유효 기간 (시작) | 사용 가능 시작일 | 2026-01-01 |
| 유효 기간 (종료) | 사용 가능 마감일 | 2026-12-31 |
| 부여 사유 | 부여 이유 | 우수사원 포상 휴가 |

3. **"부여"** 버튼을 클릭합니다.

4. 해당 직원의 잔여 연차에 즉시 반영됩니다.

### 6-2. 연차 차감

관리자가 특별한 사유로 연차를 차감해야 할 때 사용합니다.

1. **"연차 차감"** 버튼을 클릭합니다.

2. 대상 직원을 선택하고, 차감 일수와 사유를 입력합니다.

3. **"차감"** 버튼을 클릭합니다.

> **주의**: 연차 차감은 직원의 잔여 연차를 감소시키므로 신중하게 사용하세요. 차감 이력은 감사 로그에 기록됩니다.

### 6-3. 부여/차감 이력 조회

부여 및 차감 이력은 직원별로 조회할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 일시 | 부여/차감 처리 일시 |
| 유형 | 부여 / 차감 |
| 일수 | 부여 또는 차감된 일수 |
| 사유 | 부여/차감 사유 |
| 유효 기간 | 사용 가능 기간 |
| 처리자 | 처리한 관리자 이름 |

## 7. 휴가 관리대장

좌측 메뉴에서 **휴가 > 휴가관리대장** 을 클릭합니다.

전체 직원의 휴가 사용 내역을 종합적으로 조회하고 관리합니다.

### 7-1. 조회 기능

| 필터 | 설명 |
|------|------|
| 연도 | 조회할 연도 선택 |
| 부서 | 특정 부서 선택 (전체 가능) |
| 직원 | 특정 직원 검색 |
| 상태 | 승인/반려/취소 등 필터 |
| 기간 | 특정 기간 내 휴가 조회 |

### 7-2. 관리대장 표시 항목

| 항목 | 설명 |
|------|------|
| 사번 | 직원 사번 |
| 이름 | 직원 이름 |
| 부서 | 소속 부서 |
| 직급 | 직급 |
| 총 부여 | 올해 부여된 총 연차 (자동 + 수동) |
| 사용 | 승인 완료된 사용 일수 |
| 잔여 | 남은 연차 일수 |
| 사용률 | 사용일수 / 총부여 * 100 |
| 상세 | 개별 휴가 사용 내역 보기 |

### 7-3. 엑셀 내보내기

관리대장을 엑셀 파일로 다운로드할 수 있습니다.

1. 원하는 필터를 적용합니다.
2. **"엑셀 다운로드"** 버튼을 클릭합니다.
3. 현재 조회 결과가 엑셀 파일로 다운로드됩니다.

## 8. 결재선 설정

좌측 메뉴에서 **기본설정 > 권한/결재선 설정** 을 클릭합니다.

### 8-1. 결재선이란?

결재선은 휴가 또는 시간외근무 신청 시 승인을 받아야 하는 결재자의 순서를 정의합니다.

예시 (2단계 결재):
```
신청자(사원) → 1차 결재(과장) → 2차 결재(부장) → 승인 완료
```

### 8-2. 결재선 생성

1. **"결재선 추가"** 버튼을 클릭합니다.

2. 아래 항목을 입력합니다.

| 항목 | 설명 | 예시 |
|------|------|------|
| 결재선 이름 | 결재선 식별 이름 | 개발팀 휴가결재 |
| 결재선 유형 | 휴가 / 시간외근무 / 일반 | 휴가 |
| 기본 결재선 여부 | 이 결재선을 기본으로 사용할지 여부 | 예 |

3. 결재 단계를 추가합니다.

각 단계별로 설정하는 항목:

| 항목 | 설명 |
|------|------|
| 단계 순서 | 1, 2, 3... (순차적으로 처리됨) |
| 결재자 지정 방식 | 고정 지정 / 부서장 / 상위 직급자 / 전결 |
| 결재자 | 고정 지정 시 특정 직원 선택 |
| 처리 유형 | 승인 / 합의 / 통보 |

### 8-3. 결재자 지정 방식

| 방식 | 코드 | 설명 |
|------|------|------|
| 고정 지정 | FIXED | 특정 직원을 결재자로 지정 |
| 부서장 | DEPT_HEAD | 신청자 소속 부서의 장이 자동 지정 |
| 상위 직급자 | UPPER_POSITION | 신청자보다 한 단계 높은 직급자가 자동 지정 |
| 전결 | SKIP_TO_HEAD | 중간 결재를 건너뛰고 부서장이 바로 처리 |

### 8-4. 처리 유형

| 유형 | 코드 | 설명 |
|------|------|------|
| 승인 | APPROVE | 해당 단계에서 승인/반려 결정 (반려 시 전체 반려) |
| 합의 | AGREE | 의견 확인 후 다음 단계로 전달 (반려 불가) |
| 통보 | NOTIFY | 결과만 통보 (승인/반려 불필요) |

### 8-5. 결재선 예시

**2단계 결재선 (일반적)**:
| 단계 | 결재자 | 방식 | 유형 |
|------|--------|------|------|
| 1 | 직속 상관 | 상위 직급자 | 승인 |
| 2 | 부서장 | 부서장 | 승인 |

**3단계 결재선 (대규모 조직)**:
| 단계 | 결재자 | 방식 | 유형 |
|------|--------|------|------|
| 1 | 팀장 | 상위 직급자 | 승인 |
| 2 | 부서장 | 부서장 | 승인 |
| 3 | 인사팀장 | 고정 지정 | 승인 |

## 12. 보상정책 설정 (시간 지갑)

좌측 메뉴에서 **기본설정 > 보상정책 설정** 을 클릭합니다.

시간외근무 보상 방식과 휴가 차감 순서를 설정합니다.

### 12-1. 보상 방식

| 방식 | 설명 |
|------|------|
| 보상시간 적립 (대체휴무) | 연장근무 시간에 배율을 적용하여 보상시간으로 적립 |
| 수당 지급 | 연장근무에 대해 수당만 지급 (보상시간 미적립) |
| 직원 선택 가능 | 직원이 보상시간 또는 수당 중 선택 |

### 12-2. 배율 설정

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 평일 연장 배율 | 1.5배 | 평일 연장근무 보상시간 계산 배율 |
| 야간근무 배율 | 2.0배 | 야간근무 보상시간 계산 배율 |
| 휴일근무 배율 | 2.0배 | 공휴일근무 보상시간 계산 배율 |

예시: 직원이 평일 3시간 연장근무 → 3 × 1.5 = **4.5시간 보상시간 적립**

### 12-3. 시간 단위 설정

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 1일 근무시간 | 8시간 | 1일 휴가 = 8시간 차감 |
| 반차 시간 | 4시간 | 반차 = 4시간 차감 |
| 최소 사용 단위 | 1시간 | 휴가 최소 사용 단위 |

### 12-4. 차감 순서

휴가 사용 시 어떤 시간부터 차감할지 설정합니다.

| 순서 | 설명 |
|------|------|
| 보상시간 우선 → 연차 | 보상시간부터 차감 후 소진되면 연차에서 차감 |
| 연차 우선 → 보상시간 | 연차부터 차감 후 소진되면 보상시간에서 차감 |

**자동 분할 차감**: 한 종류의 시간이 부족하면 자동으로 다음 순위에서 나머지를 차감합니다.

### 12-5. 작동 흐름

1. **시간외근무 승인** → 보상시간 자동 적립 (배율 적용)
2. **연차 자동부여** → 연차시간 지갑 초기화 (일수 × 1일근무시간)
3. **휴가 승인** → 시간 지갑에서 자동 차감 (설정된 순서대로)
4. **휴가 취소** → 차감된 시간 자동 복원

## 15. 데이터 백업 및 복원

### 15-1. 자동 백업 (Docker 프로덕션 모드)

Docker Compose의 `production` 프로필을 사용하면 매일 자동으로 백업이 수행됩니다.

```bash
# 자동 백업 활성화
docker compose --profile production up -d
```

- 백업 파일 저장 위치: `msa-hr/backups/` 폴더
- 파일명 형식: `backup_20260213_030000.sql`
- 보관 기간: 30일 (30일 이상 된 백업은 자동 삭제)

### 15-2. 수동 백업

```bash
# 현재 날짜로 백업 파일 생성
docker exec msa-hr-db pg_dump -U msa msa_hr > backups/backup_$(date +%Y%m%d_%H%M%S).sql
```

> **권장**: 중요한 설정 변경 전에 반드시 수동 백업을 수행하세요.

### 15-3. 데이터 복원

```bash
# 백업 파일로 복원
docker exec -i msa-hr-db psql -U msa msa_hr < backups/backup_20260213_030000.sql
```

> **주의**: 복원 시 현재 데이터가 백업 시점의 데이터로 덮어씌워집니다. 복원 전 현재 데이터를 반드시 백업하세요.

### 15-4. 전체 초기화

모든 데이터를 삭제하고 시스템을 처음 상태로 되돌립니다.

```bash
# 1. 서비스 중지 및 데이터 볼륨 삭제
docker compose down -v

# 2. 설정 파일 삭제
rm -rf data/

# 3. 브라우저 쿠키 삭제 (setup_complete)

# 4. 시스템 재시작
docker compose up -d
```

재시작 후 Setup Wizard가 다시 표시됩니다.

### 15-5. Prisma Studio (DB 직접 관리)

개발 환경에서 데이터베이스를 직접 조회하고 수정할 수 있는 GUI 도구입니다.

```bash
npx prisma studio
```

실행 후 `http://localhost:5555` 에서 모든 테이블의 데이터를 확인하고 편집할 수 있습니다.

> **주의**: Prisma Studio에서 데이터를 직접 수정하면 시스템의 정합성이 깨질 수 있습니다. 개발/디버깅 목적으로만 사용하세요.

## 16. 외부 서비스 연동

좌측 메뉴에서 **기본설정 > 외부서비스 연동** 을 클릭합니다.

### 16-1. 구글 캘린더 연동

직원의 승인된 휴가를 구글 캘린더에 자동으로 등록합니다.

#### 사전 준비

1. Google Cloud Console (`https://console.cloud.google.com`)에서 프로젝트를 생성합니다.
2. Google Calendar API를 활성화합니다.
3. OAuth 2.0 클라이언트 ID를 생성합니다.
4. 리디렉션 URI에 `http://서버주소:3000/api/auth/google/callback`을 등록합니다.

#### 시스템 설정

1. **외부서비스 연동** 페이지에서 **"구글 캘린더"** 섹션을 찾습니다.
2. Google Cloud Console에서 발급받은 Client ID와 Client Secret을 입력합니다.
3. **"저장"** 버튼을 클릭합니다.

#### 직원별 연동

각 직원이 개별적으로 구글 계정을 연동해야 합니다.

1. 직원이 본인의 설정에서 "구글 캘린더 연동" 버튼을 클릭합니다.
2. 구글 로그인 화면이 표시됩니다.
3. 로그인 후 캘린더 접근 권한을 허용합니다.
4. 연동이 완료되면 이후 승인된 휴가가 자동으로 구글 캘린더에 등록됩니다.

#### 연동 해제

직원별로 구글 캘린더 연동을 해제할 수 있습니다.

1. 설정에서 "구글 캘린더 연동 해제" 버튼을 클릭합니다.
2. 확인 대화상자에서 "해제"를 클릭합니다.

> **참고**: 구글 캘린더 연동은 선택 기능이며, 연동하지 않아도 시스템 사용에 지장이 없습니다.

## 17. 감사 로그

시스템의 모든 주요 동작은 감사 로그(Audit Log)에 기록됩니다.

### 기록되는 항목

| 동작 | 대상 | 설명 |
|------|------|------|
| 직원 등록/수정/퇴사 | employees | 직원 정보 변경 이력 |
| 휴가 신청/취소 | leave_requests | 휴가 신청 및 취소 이력 |
| 결재 승인/반려 | approvals | 결재 처리 이력 |
| 연차 부여/차감 | leave_grants | 연차 수동 변경 이력 |
| 설정 변경 | system_configs | 시스템 설정 변경 이력 |
| 로그인/로그아웃 | sessions | 접속 이력 |

### 감사 로그 조회

관리자 메뉴에서 감사 로그를 조회할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 일시 | 동작이 수행된 날짜 및 시간 |
| 수행자 | 동작을 수행한 직원 |
| 동작 | 수행된 작업 (생성, 수정, 삭제 등) |
| 대상 | 작업 대상 (직원, 휴가 등) |
| 변경 전/후 | 데이터 변경 전후 값 (JSON) |
| IP 주소 | 접속한 IP 주소 |

### 로그 보관

감사 로그는 데이터베이스에 영구 보관되며, 별도의 삭제 기능은 제공하지 않습니다. 데이터베이스 백업에 포함되므로, 백업을 통해 장기 보관이 가능합니다.

## 부록: 자주 사용하는 관리 명령어

### Docker 관련

```bash
# 서비스 상태 확인
docker compose ps

# 로그 확인
docker compose logs -f

# DB 로그만 확인
docker compose logs -f db

# 앱 로그만 확인
docker compose logs -f app

# 서비스 재시작
docker compose restart

# 서비스 완전 종료
docker compose down

# 서비스 종료 + 데이터 삭제
docker compose down -v
```

### 데이터베이스 관련

```bash
# DB 직접 접속
docker exec -it msa-hr-db psql -U msa msa_hr

# 테이블 목록 확인 (DB 접속 후)
\dt

# 직원 목록 조회
SELECT employee_number, name, email, status FROM employees;

# 시스템 설정 조회
SELECT key, value, "group" FROM system_configs;

# DB 접속 종료
\q
```

### Prisma 관련

```bash
# 스키마 변경 후 DB에 반영
npx prisma db push

# Prisma 클라이언트 재생성
npx prisma generate

# DB GUI 도구 실행
npx prisma studio
```
