## 1. 서버 / 하드웨어 [필수]

개발 단계에서는 현재 사용 중인 Mac에서 바로 진행합니다.
운영 배포 시 아래 중 하나를 준비하세요.

| 방식 | 장점 | 단점 |
|------|------|------|
| **사내 데스크탑 PC 1대 상시 가동** | 비용 0원, 가장 간단 | PC가 꺼지면 서비스 중단 |
| **NAS (Synology 등)** | 저전력 상시 가동, Docker 지원 | NAS 구매 비용 (30~50만원) |
| **미니 PC (N100 등)** | 저전력, 소음 적음, 성능 충분 | 별도 구매 (15~30만원) |
| **클라우드 (AWS/NCP)** | 안정적, 외부 접속 쉬움 | 월 비용 발생 (3~5만원) |

**최소 사양**: CPU 2코어, RAM 4GB, SSD 20GB, Ubuntu 22.04 또는 macOS

### 확인 질문
- [ ] 어떤 서버를 사용할 건지 결정 (현재 Mac / 사내 PC / NAS / 클라우드)
- [ ] 해당 서버의 사내 고정 IP 확인 (예: `192.168.0.100`)

## 2. 외부 접속 방식 결정 [필수]

집이나 모바일에서 접속하려면 아래 중 하나를 선택해야 합니다.

### 방법 비교

| 방법 | 비용 | 난이도 | 보안 | 추천 |
|------|------|--------|------|------|
| **Cloudflare Tunnel** | 무료 | 쉬움 | 높음 | ★ 1순위 추천 |
| **Tailscale (메쉬 VPN)** | 무료 (3명까지) | 매우 쉬움 | 높음 | ★ 소규모 추천 |
| **WireGuard VPN** | 무료 | 보통 | 매우 높음 | 기술력 있으면 |
| 공유기 포트포워딩 | 무료 | 쉬움 | 낮음 (비추) | X |

### (추천) Cloudflare Tunnel - 무료, 도메인 불필요

```
[집/모바일] → HTTPS → [Cloudflare Edge] → 터널 → [사내 서버:3000]
                         (무료)              (암호화)
```

**장점:**
- 무료 (도메인 없어도 `*.cfargotunnel.com` 서브도메인 제공)
- HTTPS 자동 적용 (보안)
- 사내 방화벽/공유기 설정 변경 불필요
- 서버 IP 노출 안됨

**준비물:**
- [ ] Cloudflare 무료 계정 생성 (https://dash.cloudflare.com)
- [ ] `cloudflared` CLI 설치 (Docker 이미지로 자동 설치 예정)

### (대안) Tailscale - 무료, 가장 간단

```
[집/모바일 Tailscale] → 암호화 터널 → [사내 서버 Tailscale] → localhost:3000
```

**장점:**
- 설치만 하면 끝 (설정 거의 없음)
- 모든 기기에서 사내 IP처럼 접속
- 무료 (개인 3대, 비즈니스 무료 플랜 존재)

**준비물:**
- [ ] Tailscale 계정 생성 (https://tailscale.com)
- [ ] 서버 + 접속할 기기에 Tailscale 앱 설치

### 확인 질문
- [ ] 외부 접속 방식 결정 (Cloudflare Tunnel / Tailscale / VPN / 사내만 사용)
- [ ] 접속할 기기 수 파악 (PC, 모바일 등)

## 3. 직원 데이터 [필수]

시스템에 등록할 **직원 정보**를 엑셀이나 CSV로 준비해주세요.

### 필요한 데이터 (엑셀 양식)

| 사번 | 이름 | 부서 | 직급 | 입사일 | 이메일 | 연락처 | 상태 |
|------|------|------|------|--------|--------|--------|------|
| 1088148326 | 변인수 | 영업 | 이사 | 2020-01-02 | bis@msa.co.kr | 010-xxxx-xxxx | 재직 |
| ... | ... | ... | ... | ... | ... | ... | ... |

### 필수 항목
- [ ] **사번**: 기존 사번 체계 (유니포스트에서 사용하던 사번 그대로)
- [ ] **이름**: 한글 이름
- [ ] **부서**: 부서명 (영업, 개발 등)
- [ ] **직급**: 이사/상무/부장/차장/과장/대리/사원
- [ ] **입사일**: YYYY-MM-DD 형식 ★ 연차 자동계산의 핵심
- [ ] **이메일**: 로그인 ID로 사용 (사내 이메일 또는 개인 이메일)

### 선택 항목
- [ ] 연락처
- [ ] 프로필 사진
- [ ] 퇴사일 (퇴사자의 경우)

### 확인 질문
- [ ] 유니포스트에서 직원 데이터 엑셀 내보내기 가능한가요?
- [ ] 부서 목록 확정 (현재 영업 외 어떤 부서가 있는지)
- [ ] 직급 체계 확정 (이사-상무-부장-차장-과장-대리-사원 맞는지)

## 4. 회사 규정 확인 [필수]

### 4.1 연차 규정
현재 근로기준법 기본을 따르는 것으로 설계했습니다. 확인 필요:

- [ ] 입사 1년 미만: 매월 1일 발생 (법정 기준) → 자사 규정 동일한가요?
- [ ] 입사 1년 이상: 15일 + 2년마다 1일 추가 (최대 25일) → 동일한가요?
- [ ] 연차 사용 기한: 입사일 기준 1년? 또는 회계연도(1/1~12/31) 기준?
- [ ] 미사용 연차 이월 여부: 이월 허용? 소멸?
- [ ] 반차 기준: 오전 반차(09:00~13:00) / 오후 반차(14:00~18:00) 시간대 확인

### 4.2 휴가 유형
아래 유형 외에 추가할 것이 있는지 확인:

| 유형 | 일수 | 연차차감 | 유급 | 비고 |
|------|------|---------|------|------|
| 연차휴가 | 자동계산 | O | O | |
| 병가 | - | X | O | 진단서 필요? |
| 경조사 | 규정별 | X | O | 결혼/사망 등 일수 기준? |
| 출산휴가 | 90일 | X | O | |
| 배우자 출산 | 10일 | X | O | |
| 공가 | - | X | O | |
| 특별휴가 | - | X | O | |

- [ ] 위 유형이 맞는지 확인
- [ ] 추가 필요한 휴가 유형 있는지
- [ ] 경조사 세부 규정 (결혼 5일, 부모 사망 5일 등)

### 4.3 결재선 규정
- [ ] 기본 결재 구조: 팀장 → 부서장 → 임원? (몇 단계인지)
- [ ] 전결 규정: 누가 최종 승인 권한이 있는지
- [ ] 부서장이 휴가 낼 때 결재자는 누구인지

### 4.4 시간외근무
- [ ] 기본 근무시간: 09:00 ~ 18:00 (점심 12:00~13:00)?
- [ ] 시간외근무 사전 승인 필요 여부
- [ ] 주 최대 시간외근무 한도
