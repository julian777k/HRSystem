시스템 소개

HR SYSTEM은 중소기업을 위한 올인원 인사관리 솔루션입니다. 별도의 서버 구축이나 클라우드 비용 없이, 사내 PC 한 대에 설치하면 전 직원이 브라우저로 접속하여 사용할 수 있습니다.

이 시스템은 어떻게 동작하나요?

💡
한 줄 요약

사무실 PC 한 대에 앱을 설치하면, 그 PC가 자동으로 서버가 됩니다. 다른 직원들은 스마트폰이나 PC의 브라우저(Chrome 등)에서 주소를 입력하면 바로 사용할 수 있습니다. 별도 앱 설치는 필요 없습니다.

기존에 알고 계신 것HR SYSTEM은 이렇습니다
"클라우드 서비스처럼 매달 비용을 내야 하나요?"아닙니다. 한 번 설치하면 추가 비용이 없습니다.
"직원 전원이 앱을 설치해야 하나요?"아닙니다. PC 한 대만 설치합니다. 나머지는 브라우저로 접속합니다.
"인터넷이 안 되면 사용 못 하나요?"사용 가능합니다. 사내 네트워크(공유기)만 있으면 됩니다. 인터넷 없이도 동작합니다.
"IT 전문가가 설치해야 하나요?"아닙니다. 설치 파일을 더블클릭하면 자동으로 완료됩니다.
"데이터가 외부로 나가나요?"아닙니다. 모든 데이터는 사무실 PC 안에만 저장됩니다. 외부 서버를 사용하지 않습니다.

시스템 요구사항

2.1 서버 PC 준비하기

HR SYSTEM은 PC 한 대에 앱을 설치하면, 그 PC가 서버 역할을 합니다. 다른 직원들은 이 PC에 접속하여 사용합니다.

✅
핵심: 앱이 실행 중이어야 직원들이 접속할 수 있습니다

서버 PC에서 HR SYSTEM 앱이 켜져 있는 동안에만 다른 직원들이 접속할 수 있습니다. 최소한 근무 시간 동안은 앱이 실행된 상태를 유지해야 합니다.

💡
가장 좋은 방법: 전용 PC를 켜두기

NAS(나스 — 네트워크에 연결해 파일을 저장하는 소형 장치)처럼 한 번 설치하고, 켜둔 채로 아무도 건드리지 않는 것이 가장 편리합니다. 24시간 켜두면 야근이나 이른 출근에도 바로 접속할 수 있고, 시작프로그램에 등록해두면(5.4 참고) 재부팅 후에도 자동 실행됩니다. 여건이 안 되면 출근 시 앱을 켜고, 퇴근 시 꺼두셔도 됩니다.

어떤 PC를 서버로 사용할까요?

Intel N100 이상 / RAM 8GB / SSD 128GB 이상이면 충분합니다. 소음이 거의 없고, 전기료 월 1,000~2,000원 수준입니다. 모니터 없이 전원만 연결해두면 됩니다.

HR SYSTEM은 인터넷 속도에 영향을 주지 않습니다

HR SYSTEM이 주고받는 데이터는 직원 정보, 휴가 신청 등 텍스트 위주로, 용량이 매우 작습니다. 직원 50명이 동시에 사용해도 인터넷의 0.3% 미만만 사용하므로, 다른 업무(메일, 웹서핑, 화상회의 등)에 전혀 영향을 주지 않습니다.

3.3 NAS 설치 (Docker)

NAS(ASUSTOR, Synology, QNAP 등)를 보유하고 있다면, NAS에 HR SYSTEM을 설치할 수 있습니다. NAS는 24시간 켜져 있고, 저전력이므로 서버 용도로 가장 이상적입니다.

💡
NAS란?

NAS(Network Attached Storage)는 네트워크에 연결된 저장 장치입니다. 보통 파일 서버, 백업 장치로 사용하지만, Docker라는 기능을 지원하면 HR SYSTEM 같은 프로그램도 설치할 수 있습니다. Windows/macOS를 설치할 필요 없이, NAS의 기본 시스템(Linux) 위에서 바로 동작합니다.

NAS 설치 과정 (4단계)
1
제공 파일 2개를 NAS에 복사

아래 2개 파일을 NAS의 공유 폴더(아무 폴더)에 복사합니다.

HRInstall-0.2.0-docker(nas).zipHR SYSTEM 이미지 파일
docker-compose.yml실행 설정 파일

NAS 관리 페이지의 파일 관리자를 이용하거나, PC에서 NAS 공유 폴더로 직접 복사합니다.

2
NAS 터미널(SSH)에 접속

NAS의 터미널 또는 SSH로 접속합니다.

NAS 브랜드터미널 접속 방법
SynologyDSM → 제어판 → 터미널 및 SNMP → SSH 활성화 후, PC에서 ssh admin@NAS아이피
ASUSTORADM → 서비스 → 터미널 → SSH 활성화 후, PC에서 ssh admin@NAS아이피
QNAPQTS → 제어판 → 네트워크 및 파일 서비스 → Telnet/SSH 활성화

Windows: 명령 프롬프트에서 ssh admin@NAS아이피 입력  |  macOS: 터미널에서 동일 명령어

3
Docker 이미지 로드

파일을 복사한 폴더로 이동한 뒤, 아래 명령어를 입력합니다.

# 파일이 있는 폴더로 이동
cd /volume1/docker   # (경로는 NAS마다 다름)

# Docker 이미지 로드
docker load -i HRInstall-0.2.0-docker(nas).zip

약 1~2분 소요됩니다. "Loaded image: hr-system:0.2.0" 메시지가 나오면 성공입니다.

4
서비스 시작

같은 폴더에서 아래 명령어로 서비스를 시작합니다.

docker-compose up -d

"Creating hr-system ... done" 메시지가 나오면 성공입니다. 이후 NAS를 재시작해도 자동으로 실행됩니다.

5
브라우저에서 초기 설정

PC 또는 스마트폰의 브라우저에서 아래 주소에 접속합니다.

http://NAS아이피:3000/setup
예시: http://192.168.1.100:3000/setup

초기 설정 화면이 나타나면, Windows/macOS와 동일하게 회사 정보와 관리자 계정을 입력합니다. 설정 완료 후 4. 초기 설정과 동일한 절차입니다.

✅
NAS의 장점

NAS는 24시간 켜져 있으므로 별도의 시작프로그램 설정이 필요 없습니다. 전기료도 월 1,000~3,000원 수준이며, NAS를 재부팅해도 HR SYSTEM이 자동으로 다시 시작됩니다. 데이터는 NAS 내부에 안전하게 저장됩니다.

NAS의 IP 주소 확인 방법

NAS 브랜드확인 방법
SynologyDSM → 제어판 → 네트워크 → 네트워크 인터페이스에서 확인
또는 find.synology.com 접속
ASUSTORAiMaster 앱(스마트폰)에서 확인
또는 ADM → 설정 → 네트워크에서 확인
QNAPQfinder Pro(PC 프로그램)에서 확인
또는 QTS → 제어판 → 네트워크에서 확인
공통공유기 관리 페이지(보통 192.168.0.1)에서 연결된 장치 목록 확인

NAS 서비스 관리 명령어

작업명령어
서비스 시작docker-compose up -d
서비스 중지docker-compose down
서비스 재시작docker-compose restart
실행 상태 확인docker-compose ps
로그 확인docker-compose logs
⚠️
NAS에서 삭제하는 경우

서비스 중지(docker-compose down) 후 이미지 삭제(docker rmi hr-system:0.2.0)를 실행합니다. 데이터는 Docker 볼륨에 별도 보관되므로, 볼륨을 삭제(docker volume rm 폴더명_hr-data)하지 않는 한 유지됩니다.

5.1 서버 PC에서 접속

앱을 실행하면 자동으로 HR SYSTEM이 표시됩니다. 별도 조작이 필요 없습니다.

💡
접속 주소는 자동으로 관리됩니다

앱이 실행될 때마다 현재 네트워크 주소를 자동 감지하여 하단 바에 표시합니다. PC를 재부팅해도 앱이 자동 실행되면 새 주소가 표시됩니다. 직원에게는 하단 바의 주소를 안내하면 됩니다.

✅
절전 모드 자동 방지

HR SYSTEM이 실행 중이면 서버 PC가 자동으로 절전/수면 모드에 들어가지 않습니다. 화면이 꺼지는 것은 정상이며, 서버는 계속 동작합니다. 앱을 종료하면 절전 모드가 다시 정상적으로 작동합니다.

⚠️
서버 PC에서 앱이 실행 중이어야 합니다

HR SYSTEM 앱이 켜져 있는 동안에만 다른 기기에서 접속할 수 있습니다. PC를 끄거나 앱을 종료하면 접속이 불가합니다. 최소한 근무 시간 동안은 앱을 켜두세요.

6.1 로그인 / 회원가입

로그인

회원가입

1. 로그인 화면에서 "회원가입" 링크를 클릭합니다.
2. 이름, 이메일, 비밀번호, 전화번호, 부서, 직급을 입력합니다.

예산 관리

8

보안

HR SYSTEM은 직원 개인정보와 인사 데이터를 안전하게 보호하기 위해 다양한 보안 기능을 내장하고 있습니다.

비밀번호 보안

보안 항목내용
비밀번호 암호화PBKDF2 알고리즘으로 비밀번호를 암호화하여 저장합니다. 원본 비밀번호는 어디에도 저장되지 않습니다.
최소 길이비밀번호는 최소 8자 이상이어야 합니다.
최대 길이 제한비밀번호는 최대 128자로 제한됩니다. (비정상적으로 긴 비밀번호에 의한 서비스 거부 공격 방지)
초기 비밀번호 변경슈퍼관리자는 최초 로그인 시 반드시 비밀번호를 변경해야 합니다.

로그인 보안

보안 항목내용
로그인 속도 제한짧은 시간 내 반복적인 로그인 시도를 제한하여 무차별 대입 공격(비밀번호를 무작위로 대입하는 공격)을 차단합니다.
JWT 토큰 인증로그인 후 JWT(JSON Web Token) 기반 인증을 사용합니다. 일반 사용자와 슈퍼관리자는 별도의 JWT 비밀키로 분리되어 보안이 강화됩니다.
세션 만료일정 시간 사용하지 않으면 자동으로 로그아웃되어 무단 접근을 방지합니다.

웹 보안 헤더

HR SYSTEM은 최신 웹 보안 표준에 따른 보안 헤더를 자동으로 적용합니다.

보안 헤더기능
CSP (Content Security Policy)허가되지 않은 외부 스크립트 실행을 차단하여 XSS(크로스사이트 스크립팅) 공격을 방지합니다.
HSTSHTTPS 연결을 강제하여 통신 내용이 도청되는 것을 방지합니다.
X-Frame-Options다른 사이트에서 HR SYSTEM을 프레임으로 삽입하는 것을 차단합니다. (클릭재킹 방지)
X-Content-Type-Options브라우저의 MIME 타입 추측을 방지하여 파일 기반 공격을 차단합니다.

슈퍼관리자 보안

시스템 최고 관리자(슈퍼관리자)에 대한 추가 보안 기능이 적용됩니다.

1
별도 인증 체계

슈퍼관리자는 일반 사용자와 별도의 로그인 경로(/super-admin/login)와 별도의 JWT 비밀키를 사용합니다.

2
초기 비밀번호 강제 변경

슈퍼관리자가 최초 로그인하면 반드시 비밀번호를 변경해야 합니다. 기본 비밀번호를 그대로 사용할 수 없습니다.

3
테넌트(회사) 관리

슈퍼관리자 패널에서 회사(테넌트) 생성, 정지, 삭제 및 사용 현황 모니터링이 가능합니다.

🔒
보안 권장 사항

• 관리자 비밀번호는 12자 이상의 강력한 비밀번호를 사용하세요.
• 주기적으로 비밀번호를 변경하는 것을 권장합니다.
• 외부 접속 설정 시 반드시 HTTPS를 사용하세요. (12. 고급 설정 참고)
• 퇴직 직원의 계정은 즉시 비활성화하세요.

9

데이터 백업 및 복원

⚠️
정기 백업은 필수입니다

HR SYSTEM의 모든 데이터(직원 정보, 휴가 내역, 결재 기록 등)는 서버 PC의 하나의 파일(data.db)에 저장됩니다. 이 파일만 안전하게 보관하면 언제든 데이터를 복원할 수 있습니다.

9.1 시스템 트레이에서 백업 (권장)

가장 간편한 방법입니다. 클릭 몇 번으로 백업이 완료됩니다. 시스템 트레이란 화면 우측 하단 시계 옆의 아이콘 모음 영역입니다.

1
시스템 트레이 아이콘 찾기

화면 우측 하단 작업표시줄에서 HR SYSTEM 아이콘을 찾습니다. 아이콘이 보이지 않으면 ▲ (숨겨진 아이콘 표시) 버튼을 클릭하세요.

2
트레이 아이콘 우클릭

HR SYSTEM 트레이 아이콘을 마우스 오른쪽 버튼으로 클릭하면 메뉴가 나타납니다.

3
"DB 백업" 클릭

메뉴에서 DB 백업을 클릭합니다.

4
저장 위치 선택

파일 저장 대화상자가 나타납니다. USB 메모리, 외장하드, 또는 원하는 폴더를 선택합니다.
파일명은 자동으로 hr-backup-2026-02-17.db 형식으로 생성됩니다.

5
저장 버튼 클릭

저장 버튼을 누르면 "백업이 완료되었습니다" 메시지가 나타납니다. 완료!

✅
백업 파일 보관 팁

USB 메모리 또는 외장하드에 저장하면 서버 PC가 고장나더라도 데이터를 복원할 수 있습니다. 서버 PC 내부에만 저장하면 PC 고장 시 백업도 함께 유실될 수 있습니다.

9.2 파일 직접 복사로 백업

시스템 트레이를 사용할 수 없는 경우, 데이터베이스 파일을 직접 복사할 수 있습니다.

데이터베이스 파일 위치

운영체제파일 경로
WindowsC:\Users\[사용자명]\AppData\Roaming\hr-system\data.db
macOS~/Library/Application Support/hr-system/data.db
NAS (Docker)Docker 볼륨 내부 — docker cp hr-system:/app/data/data.db ./backup-data.db 명령어로 복사
💡
AppData 폴더가 안 보여요?

Windows의 AppData(앱 데이터 저장) 폴더는 기본적으로 숨겨져 있습니다. 아래 방법으로 열 수 있습니다:
1. Win + R 키를 눌러 실행 창을 엽니다.
2. %APPDATA%\hr-system 을 입력하고 확인을 누릅니다.
3. 열린 폴더에서 data.db 파일을 찾아 복사합니다.

1
HR SYSTEM 앱 종료

백업의 정확성을 위해 앱을 먼저 종료하는 것을 권장합니다. (시스템 트레이 → 우클릭 → "종료")

2
데이터 폴더 열기

Win + R → %APPDATA%\hr-system 입력 → 확인

3
data.db 파일 복사

data.db 파일을 선택하고 Ctrl + C로 복사한 후, USB 메모리나 외장하드에 Ctrl + V로 붙여넣기합니다.

9.3 백업 파일로 복원하기

백업해 둔 파일로 데이터를 되돌리는 방법입니다.

⚠️
복원하면 현재 데이터가 덮어씌워집니다

복원을 진행하면 현재 시스템의 데이터가 백업 시점의 데이터로 교체됩니다. 복원 전에 현재 데이터도 별도로 백업해 두는 것을 권장합니다.

1
HR SYSTEM 앱 완전 종료

시스템 트레이에서 HR SYSTEM 아이콘 우클릭 → "종료"를 클릭합니다. 앱이 완전히 종료되어야 합니다.

2
데이터 폴더 열기

Win + R → %APPDATA%\hr-system 입력 → 확인

3
현재 data.db를 다른 이름으로 변경 (안전장치)

현재 data.db 파일을 data-old.db 등으로 이름을 변경합니다. (파일 선택 → F2 → 새 이름 입력)

4
백업 파일 복사 + 이름 변경

USB 메모리나 외장하드에서 백업 파일(예: hr-backup-2026-02-17.db)을 이 폴더에 복사한 후, 파일 이름을 data.db로 변경합니다.

5
앱 다시 실행

HR SYSTEM을 실행하면 복원된 데이터로 시스템이 작동합니다.

자주 묻는 질문 (FAQ)

서버 PC를 꺼도 되나요?
앱이 실행 중일 때만 다른 기기에서 접속할 수 있습니다. 최소한 근무 시간 동안은 켜두세요. 24시간 켜두면 더 편리합니다. 5.4 시작프로그램 등록을 참고하여 자동 실행을 설정하면, PC를 재시작해도 앱이 자동으로 실행됩니다.
여러 대에 앱을 설치하면 데이터가 공유되나요?
아닙니다. 각 PC에 설치하면 별도의 독립된 시스템이 됩니다. 데이터를 공유하려면 한 대만 설치하고 나머지는 브라우저로 접속해야 합니다.
방화벽 설정을 따로 해야 하나요?
아닙니다. 설치 과정에서 Windows 방화벽이 자동으로 설정됩니다. 어떤 네트워크 환경(유선, WiFi, 공용, 사설)에서든 별도 조작 없이 다른 기기에서 바로 접속할 수 있습니다.
스마트폰에서 접속 주소를 어떻게 알 수 있나요?
서버 PC의 앱 하단 바에 표시된 QR 코드 썸네일을 클릭하면 큰 QR 코드가 나타납니다. 스마트폰 카메라로 이 QR 코드를 스캔하면 자동으로 접속됩니다. 화면 우클릭 → "모바일 접속 QR"로도 열 수 있습니다.
데이터 백업은 어떻게 하나요?
가장 간편한 방법은 시스템 트레이 아이콘 우클릭 → "DB 백업"을 이용하는 것입니다. 자세한 방법은 9. 데이터 백업 및 복원 섹션을 참고하세요.
IP 주소가 바뀌었어요.
PC 이름 주소(예: http://MY-PC:3000)로 접속하는 경우에는 IP가 바뀌어도 영향이 없습니다. QR 코드를 사용하는 스마트폰 사용자는, 앱을 재실행하면 새 IP가 자동 감지되어 QR 코드도 자동 갱신되므로 다시 스캔하면 됩니다.
외부(집, 외근지)에서 접속할 수 있나요?
기본적으로 같은 네트워크(같은 공유기) 내에서만 접속 가능합니다. 외부 접속이 필요한 경우 VPN(가상 사설 네트워크) 또는 Cloudflare Tunnel 등 추가 설정이 필요합니다. 자세한 내용은 12. 고급 설정 (외부 접속)을 참고하거나 별도 문의해주세요.
동시 접속자 수 제한이 있나요?
소프트웨어 상의 제한은 없습니다. 서버 PC의 성능에 따라 수십 명까지 동시 접속이 가능합니다.
11

문제 해결

증상해결 방법
앱 실행 후 화면이 안 나옴잠시 기다려주세요 (최초 실행 시 10~20초 소요). 계속 안 되면 앱을 종료 후 재실행합니다.
다른 기기에서 접속 안 됨① 서버 PC와 같은 네트워크(같은 공유기)에 연결되어 있는지 확인 ② 하단 바의 접속 주소가 정확한지 확인 ③ 서버 PC가 켜져 있고 앱이 실행 중인지 확인 ④ 앱을 재실행해 봅니다
하단 바가 안 보여요하단 바의 × 버튼을 누른 경우 페이지를 새로고침(F5)하면 다시 나타납니다.
QR 코드가 안 나와요서버 PC가 공유기에 연결되어 있는지 확인합니다. 네트워크(LAN/WiFi) 연결이 되어야 QR 코드가 생성됩니다. 인터넷은 필요하지 않습니다. QR 코드가 안 될 경우, 하단 바의 주소를 직접 입력하여 접속할 수 있습니다.
로그인은 되는데 화면 안 바뀜브라우저의 캐시(임시 저장 데이터)를 삭제 후 재접속합니다. (Ctrl+Shift+Delete)
비밀번호를 잊었어요관리자에게 비밀번호 초기화를 요청합니다. 관리자는 직원 관리에서 비밀번호를 재설정할 수 있습니다.
macOS에서 "손상된 파일" 오류터미널에서 xattr -cr /Applications/HR\ SYSTEM.app 실행 후 다시 엽니다.
데이터가 사라졌어요앱 삭제/재설치 시 데이터가 초기화될 수 있습니다. 9. 데이터 백업 및 복원을 참고하여 정기 백업을 권장합니다. 백업 파일이 있다면 복원이 가능합니다.
12

고급 설정 (외부 접속)

⚠️
이 섹션은 고급 사용자를 위한 참고 자료입니다

HR SYSTEM은 같은 네트워크(사무실 내부) 전용으로 설계되었습니다. 외부에서 접속하는 것은 보안 위험이 있으므로, 특별한 이유가 없다면 사용하지 않는 것을 권장합니다.

외부 접속이 필요한 경우

재택근무, 출장 중, 또는 다른 지점에서 HR SYSTEM에 접속해야 하는 경우입니다. 아래 방법들은 모두 추가 설정과 유지 비용이 필요합니다.

방법 1: Cloudflare Tunnel (권장)

항목내용
원리서버 PC에 Cloudflare 프로그램을 설치하면, Cloudflare가 안전한 터널(암호화된 통로)을 만들어 외부에서 접속 가능하게 함
장점서버 PC에만 설치, 공유기 설정 불필요, HTTPS(암호화 접속) 자동 적용, 보안 우수
비용기본 무료 / 도메인(인터넷 주소, 예: mycompany.com) 구매 필요 (연 1~2만원)
난이도중급 (Cloudflare 계정 생성 + 도메인 설정 + 터널 설정)
보안Cloudflare의 DDoS 방어(대량 접속 공격 차단), WAF(웹 방화벽) 등 보안 기능 포함

방법 2: DDNS + 포트 포워딩

항목내용
원리공유기에서 외부 포트(통신 통로)를 내부 서버 PC로 연결(포트 포워딩) + 변하는 외부 IP 주소를 도메인(인터넷 주소)에 자동 연결(DDNS — 동적 도메인 서비스)
장점무료 가능 (무료 DDNS 서비스 사용 시)
비용무료 ~ 연 1만원 (DDNS 서비스에 따라)
난이도중~상급 (공유기 관리자 페이지 접속 + 포트 포워딩 설정 + DDNS 설정)
보안취약 — 포트(통신 통로)가 인터넷에 직접 노출되므로 해킹 위험 있음
🚫
DDNS + 포트 포워딩의 보안 위험

이 방식은 서버 PC의 포트(통신 통로)를 인터넷에 직접 노출합니다. 비밀번호 무차별 대입 공격, 포트 스캔(열린 통로 탐색), 취약점 악용 등의 위험이 있습니다. 반드시 강력한 비밀번호를 설정하고, 가능하면 VPN(가상 사설 네트워크) 또는 Cloudflare Tunnel을 대신 사용하세요.
