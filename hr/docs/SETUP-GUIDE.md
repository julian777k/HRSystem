## 2. Docker Desktop 설치

Docker Desktop은 데이터베이스(PostgreSQL)를 자동으로 실행해주는 프로그램입니다. 별도로 데이터베이스를 설치하거나 관리할 필요가 없습니다.

### macOS에서 설치

1. **Docker Desktop 다운로드 페이지에 접속합니다.**
   - 브라우저에서 `https://www.docker.com/products/docker-desktop/` 로 이동합니다.

2. **"Download for Mac" 버튼을 클릭합니다.**
   - Apple Silicon (M1/M2/M3/M4) Mac을 사용 중이라면 **"Mac with Apple Silicon"** 을 선택합니다.
   - Intel Mac을 사용 중이라면 **"Mac with Intel chip"** 을 선택합니다.
   - 본인의 Mac 종류를 모르겠다면, 화면 좌측 상단의 Apple 메뉴 > "이 Mac에 관하여"를 클릭하여 확인할 수 있습니다. "칩" 항목에 "Apple M1" 등이 표시되면 Apple Silicon이고, "프로세서" 항목에 "Intel"이 표시되면 Intel Mac입니다.

3. **다운로드된 `.dmg` 파일을 더블클릭합니다.**
   - Docker 아이콘을 Applications 폴더로 드래그 앤 드롭합니다.

4. **Applications 폴더에서 Docker를 실행합니다.**
   - Launchpad 또는 Finder > Applications에서 Docker를 찾아 더블클릭합니다.
   - 처음 실행 시 시스템 권한을 요청할 수 있습니다. "허용" 또는 비밀번호를 입력하세요.

5. **Docker Desktop이 실행되면 화면 상단 메뉴바에 고래 모양 아이콘이 나타납니다.**
   - 고래 아이콘이 움직이지 않고 고정되면 Docker가 준비된 것입니다.
   - 보통 1~2분 정도 소요됩니다.

### Windows에서 설치

1. Docker Desktop 다운로드 페이지(`https://www.docker.com/products/docker-desktop/`)에 접속합니다.
2. "Download for Windows" 버튼을 클릭합니다.
3. 다운로드된 설치 파일을 실행하고 안내에 따라 설치를 완료합니다.
4. 설치 완료 후 Docker Desktop을 실행합니다.
5. WSL 2 업데이트가 필요하다는 메시지가 나오면 안내에 따라 업데이트합니다.

### Docker 설치 확인

터미널(Mac) 또는 명령 프롬프트(Windows)를 열고 아래 명령어를 입력합니다.

```bash
docker --version
```

`Docker version 27.x.x` 와 같이 버전이 출력되면 정상적으로 설치된 것입니다.

1. 전달받은 `msa-hr` 폴더를 원하는 위치에 복사합니다.

```bash
# 예시: 바탕화면에 복사한 경우
cd ~/Desktop/msa-hr
```

## 4. 시스템 시작

### 방법 1: Docker Compose로 전체 시작 (권장)

아래 명령어 하나로 데이터베이스와 시스템이 모두 실행됩니다.

```bash
docker compose up -d
```

이 명령어를 실행하면 다음이 자동으로 수행됩니다.
- PostgreSQL 데이터베이스 생성 및 실행
- MSA HR 웹 애플리케이션 실행

약 1~3분 후 브라우저에서 접속할 수 있습니다.

### 방법 2: 개발 모드로 시작

개발자가 테스트 목적으로 사용하는 방법입니다.

```bash
# 1. 데이터베이스만 Docker로 실행
docker compose up -d db

# 2. 애플리케이션을 개발 모드로 실행
npm run dev
```

### 접속 확인

시스템이 시작되면 웹 브라우저를 열고 아래 주소로 접속합니다.

```
http://localhost:3000
```

- 같은 네트워크의 다른 컴퓨터에서 접속하려면 서버 컴퓨터의 IP 주소를 사용합니다.
  - 예: `http://192.168.0.100:3000`
  - Mac에서 IP 주소 확인: 시스템 설정 > 네트워크에서 확인 가능

## 5. Setup Wizard로 초기 설정

처음 접속하면 자동으로 **Setup Wizard(초기 설정 마법사)** 가 표시됩니다. 5단계를 따라 시스템을 설정합니다.

### 5-1. 1단계: 데이터베이스 설정

데이터베이스 연결 정보를 입력하는 단계입니다. **Docker Compose로 실행한 경우 아래 기본값을 그대로 사용하면 됩니다.**

| 항목 | 기본값 | 설명 |
|------|--------|------|
| DB 호스트 | `localhost` | 그대로 유지 |
| DB 포트 | `5432` | 그대로 유지 |
| DB 사용자 | `msa` | 그대로 유지 |
| DB 비밀번호 | `msa_password` | docker-compose.yml에 설정된 비밀번호 입력 |
| DB 이름 | `msa_hr` | 그대로 유지 |

입력 후 **"연결 테스트"** 버튼을 클릭합니다.

- "연결 성공" 메시지가 나타나면 **"다음"** 을 클릭합니다.
- "연결 실패" 메시지가 나타나면 Docker가 정상 실행 중인지 확인하세요.

> **참고**: Docker Compose를 사용하는 경우, DB 비밀번호 항목에 `msa_password`를 입력하세요. 이 비밀번호는 `docker-compose.yml` 파일에 설정되어 있습니다.

### 5-2. 2단계: 회사 정보

회사의 기본 정보를 입력합니다.

| 항목 | 예시 | 설명 |
|------|------|------|
| 회사명 | (주)엠에스에이 | 시스템에 표시될 회사명 |
| 기본 근무시간 (시작) | 09:00 | 출근 시간 |
| 기본 근무시간 (종료) | 18:00 | 퇴근 시간 |
| 점심시간 (시작) | 12:00 | 점심 시작 시간 |
| 점심시간 (종료) | 13:00 | 점심 종료 시간 |
| 서버 접속 URL | http://localhost:3000 | 시스템 접속 주소. 사내 IP로 접속할 경우 `http://192.168.x.x:3000` 형태로 변경 |

입력 후 **"다음"** 을 클릭합니다.

### 5-3. 3단계: 관리자 계정

시스템의 최초 관리자(시스템 관리자) 계정을 생성합니다. 이 계정으로 다른 직원을 등록하고 시스템을 관리하게 됩니다.

| 항목 | 예시 | 설명 |
|------|------|------|
| 사번 | 2024001 | 관리자의 사번 |
| 이름 | 홍길동 | 관리자 이름 |
| 이메일 (로그인 ID) | admin@msa.co.kr | 로그인 시 사용할 이메일 주소 |
| 비밀번호 | ******** | 8자 이상의 비밀번호 |
| 비밀번호 확인 | ******** | 위와 동일한 비밀번호 재입력 |
| 부서 | 경영지원 | 관리자가 소속된 부서 (기본: 경영지원) |
| 직급 | 이사 | 드롭다운에서 선택 (이사/상무/부장/차장/과장/대리/사원) |

> **중요**: 관리자 이메일과 비밀번호는 반드시 기억해 두세요. 이 정보로 시스템에 로그인합니다.

입력 후 **"다음"** 을 클릭합니다.

### 5-4. 4단계: 기본 규정 설정

연차 및 결재 관련 기본 규정을 설정합니다. 나중에 관리자 메뉴에서 변경할 수 있으므로, 우선 대략적으로 설정해도 됩니다.

#### 연차 기준

- **입사일 기준**: 직원 개인의 입사일을 기준으로 연차가 부여됩니다. (예: 3월 입사자는 매년 3월에 연차 발생)
- **회계연도 기준**: 매년 1월 1일을 기준으로 전 직원에게 연차가 부여됩니다.

#### 반차 시간 설정

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 오전 반차 시작 | 09:00 | 오전 반차 사용 시 근무 시작 시간 |
| 오전 반차 종료 | 13:00 | 오전 반차 사용 시 출근 시간 |
| 오후 반차 시작 | 14:00 | 오후 반차 사용 시 퇴근 시간 |
| 오후 반차 종료 | 18:00 | 오후 반차 사용 시 근무 종료 시간 |

#### 기본 결재 단계

- **1단계**: 직속 상관 1명의 승인만 필요
- **2단계**: 직속 상관 + 부서장 등 2명의 순차 승인 필요 (권장)
- **3단계**: 3명의 순차 승인 필요

#### 미사용 연차

- **이월**: 사용하지 않은 연차가 다음 해로 이월됩니다.
- **소멸**: 사용하지 않은 연차는 해당 연도 말에 소멸됩니다.

입력 후 **"다음"** 을 클릭합니다.

### 5-5. 5단계: 설정 완료

지금까지 입력한 모든 설정 내용이 요약 표시됩니다.

- **데이터베이스**: 연결 정보 확인
- **회사 정보**: 회사명, 근무시간 확인
- **관리자 계정**: 이름, 이메일, 부서/직급 확인
- **기본 규정**: 연차 기준, 반차 시간, 결재 단계, 미사용 연차 정책 확인

모든 정보가 올바른지 확인한 후 **"시스템 시작"** 버튼을 클릭합니다.

시스템 시작 과정에서 수행되는 작업:
1. 데이터베이스 테이블 자동 생성 (30개 이상)
2. 기본 직급 데이터 생성 (이사, 상무, 부장, 차장, 과장, 대리, 사원)
3. 기본 부서 생성
4. 관리자 계정 생성
5. 기본 휴가 종류 생성 (연차, 병가, 경조사, 출산휴가, 배우자출산, 공가, 특별휴가)
6. 시간외근무 정책 설정
7. 보상정책(Compensation Policy) 초기 설정
8. 시간 지갑(Time Wallet) 시스템 초기화
9. 시스템 환경 변수 자동 저장

완료되면 자동으로 로그인 페이지로 이동합니다.

## 6. 로그인 확인

Setup Wizard 완료 후 로그인 페이지가 표시됩니다.

1. **이메일** 항목에 3단계에서 입력한 관리자 이메일을 입력합니다.
2. **비밀번호** 항목에 3단계에서 설정한 비밀번호를 입력합니다.
3. **"로그인"** 버튼을 클릭합니다.

로그인에 성공하면 권한에 따라 화면이 표시됩니다.
   - **관리자** (SYSTEM_ADMIN, COMPANY_ADMIN): 관리자 대시보드(`/admin`)로 이동
   - **일반 직원** (DEPT_ADMIN, BASIC): 직원 대시보드(`/dashboard`)로 이동

## 7. 시스템 종료 및 재시작

### 시스템 종료

```bash
# 전체 서비스 종료
docker compose down
```

> **참고**: `docker compose down` 명령은 컨테이너만 종료하며, 데이터베이스의 데이터는 보존됩니다. 데이터는 Docker 볼륨(`pgdata`)에 안전하게 저장되어 있습니다.

### 시스템 재시작

```bash
# 전체 서비스 재시작
docker compose up -d
```

재시작 후에는 Setup Wizard가 다시 표시되지 않으며, 바로 로그인 화면이 나타납니다.

### Docker Desktop 확인

Docker Desktop 애플리케이션을 열면 실행 중인 컨테이너 목록을 확인할 수 있습니다.

| 컨테이너 | 용도 |
|----------|------|
| msa-hr-db | PostgreSQL 데이터베이스 |
| msa-hr-app | MSA HR 웹 애플리케이션 |

각 컨테이너 옆의 재생/정지 버튼으로도 개별 제어가 가능합니다.

## 8. 문제 해결 (FAQ)

### Q1. "연결 실패" 메시지가 나와요

**원인**: Docker에서 PostgreSQL이 아직 시작되지 않았거나, Docker Desktop이 실행되지 않은 상태입니다.

**해결 방법**:
1. Docker Desktop이 실행 중인지 확인합니다 (메뉴바에 고래 아이콘 확인).
2. 터미널에서 다음 명령어로 컨테이너 상태를 확인합니다.
   ```bash
   docker compose ps
   ```
3. `msa-hr-db`가 `running` 상태인지 확인합니다.
4. 컨테이너가 실행되지 않았다면 다시 시작합니다.
   ```bash
   docker compose up -d db
   ```
5. PostgreSQL이 완전히 시작되려면 약 10~30초가 걸릴 수 있습니다. 잠시 기다린 후 다시 시도하세요.

### Q2. "localhost:3000"에 접속이 안 돼요

**원인**: 웹 애플리케이션이 아직 시작되지 않았습니다.

**해결 방법**:
1. Docker Compose로 실행한 경우:
   ```bash
   docker compose ps
   ```
   `msa-hr-app`이 `running` 상태인지 확인합니다.

2. 개발 모드로 실행한 경우:
   ```bash
   npm run dev
   ```
   터미널에 `Ready` 메시지가 나타날 때까지 기다립니다.

3. 포트 3000이 다른 프로그램에서 사용 중일 수 있습니다. 터미널에서 확인:
   ```bash
   lsof -i :3000
   ```

### Q3. Setup Wizard를 다시 실행하고 싶어요

Setup Wizard는 최초 1회만 자동으로 표시됩니다. 다시 실행하려면:

1. 브라우저의 쿠키를 삭제합니다.
   - Chrome: 주소창 왼쪽 자물쇠 아이콘 클릭 > 쿠키 및 사이트 데이터 > 삭제
2. 또는 브라우저에서 직접 `http://localhost:3000/setup` 으로 접속합니다.

> **주의**: Setup Wizard를 다시 실행하면 기존 설정이 덮어씌워질 수 있습니다.

### Q4. 비밀번호를 잊었어요

관리자 비밀번호를 잊은 경우, 데이터베이스에서 직접 재설정해야 합니다. 담당 개발자에게 문의하세요.

### Q5. 다른 컴퓨터에서 접속하려면?

MSA HR 시스템이 실행 중인 컴퓨터(서버)의 IP 주소를 확인한 후, 같은 네트워크에 연결된 다른 컴퓨터에서 `http://서버IP:3000` 으로 접속합니다.

**서버 IP 확인 방법 (Mac)**:
1. 시스템 설정 > 네트워크를 엽니다.
2. 연결된 네트워크(Wi-Fi 또는 이더넷)를 클릭합니다.
3. IP 주소 항목을 확인합니다 (예: 192.168.0.100).

**서버 IP 확인 방법 (터미널)**:
```bash
# Mac
ifconfig | grep "inet " | grep -v 127.0.0.1

# Windows
ipconfig
```

### Q6. Docker Desktop에 "Engine stopped" 또는 "Docker Desktop starting..."이 계속 떠요

**해결 방법**:
1. Docker Desktop을 완전히 종료합니다.
2. Mac: Docker Desktop 앱을 오른쪽 클릭 > "Quit Docker Desktop"
3. 다시 Docker Desktop을 실행합니다.
4. 그래도 해결되지 않으면, Docker Desktop을 삭제 후 재설치합니다.

### Q7. 데이터를 백업하고 싶어요

데이터베이스 백업 방법:

```bash
# 백업 파일 생성
docker exec msa-hr-db pg_dump -U msa msa_hr > backup_$(date +%Y%m%d).sql
```

백업 파일은 현재 폴더에 `backup_20260213.sql` 과 같은 이름으로 저장됩니다.

**복원 방법**:
```bash
docker exec -i msa-hr-db psql -U msa msa_hr < backup_20260213.sql
```

### Q8. 시스템을 완전히 초기화하고 싶어요

> **주의**: 이 작업은 모든 데이터를 삭제합니다. 반드시 백업 후 진행하세요.

```bash
# 1. 서비스 중지 및 볼륨 삭제
docker compose down -v

# 2. 설정 파일 삭제
rm -rf data/

# 3. 다시 시작
docker compose up -d
```

이후 `http://localhost:3000` 에 접속하면 Setup Wizard가 다시 표시됩니다.

### Q9. 포트 5432가 이미 사용 중이라고 나와요

컴퓨터에 이미 PostgreSQL이 설치되어 있는 경우 포트가 충돌할 수 있습니다.

**해결 방법**:
1. 기존 PostgreSQL을 종료합니다.
2. 또는 `docker-compose.yml` 파일에서 포트를 변경합니다 (예: `"5433:5432"`). 이 경우 Setup Wizard에서 DB 포트도 `5433`으로 변경해야 합니다.
